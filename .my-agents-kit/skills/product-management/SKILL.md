## Core Principles

1.  **INVEST Criteria** for User Stories:
    - **I**ndependent
    - **N**egotiable
    - **V**aluable
    - **E**stimable
    - **S**mall
    - **T**estable

2.  **The "Three Amigos"**:
    - **Business** (Product Owner): What problem are we solving?
    - **Development** (Engineer): How do we build it?
    - **Testing** (QA): What could go wrong?

3.  **Definition of Done (DoD)**:
    - Code written?
    - Tested?
    - Documented?
    - Accepted by PO?

## Skill Usage

Use this skill when:
- converting vague "ideas" into concrete specs.
- breaking down Epics into Stories.
- dealing with scope creep (use Prioritization).
- defining "Acceptance Criteria" for developers.
